CSS Positioning

What are the differences between relative, absolute, fixed, and static positioning?

One of the big struggles I had this week was learning about the different positioning types and how to use them, so for my blog this week, I decided I'd give a shot to making positioning clear. Here's my attempt.

First I think it's important to understand why this topic is even important. When thinking about any given page, a great way to start thinking about positioning is imagining a bunch of boxes floating around in space. They can be above, below, to the right or left of each other, or they can be on top of each other. Positioning is the way one would arrange their boxes so that they behave exactly how you'd expect. Surprisingly to me, this was actually pretty difficult.

  • Static: I think it makes sense to start with static, since it's the default value (if no other position value is specified.
    • For a given box in a sea of boxes, if there's no position "property" specified, the box will just go where it naturally goes in relation to everything else.
    • This could mean that it gets pushed to the right by another box, or gets bumped to another row.
    • It has a pretty low claim to occupy the space its in. Others can trump it. It just means that the box will show up where it's supposed to.
  • Relative: Next in line is relative. If static is where a box is supposed to go, relative is where it's supposed to go -- plus or minus a few adjustments.
    • When relative is selected as the position type, we now have the power to say things like "left: 50px" which means that it will get bumped from the left of its static position by 50 pixels.
    • I think the best explanation I read for this was that its just like static, but with adjustments
  • Absolute: Abslolute begins to have some power. An absolute position goes where it wants to go.
    • The key element that's confusing about absolute is that it's still relative to other objects. In particular, it's relative to the first parent box.
    • So I think two examples can make this clear.
      • Example 1: If a box has a parent box with positioning (that is, it's inside another box that has a position specicied) and you specify absolute positioning, the box will move relative to the parent box. So if you again say "left: 50px' but the parent box is already 50px left from the side of the page, the box will now be at 100px.
      • Example 2: However, if the parent box doesn't have positioning, or the box being specified as absolute does not have a parent box, it will just be 50px from the left of the entire screen.
  • Fixed: Finally, fixed! Fixed is perhaps the easiest to understand. It just goes where it wants all the time and stays there.
    • If we specify a box as fixed with "left: 50px" - the box will basically be 50px from the left side of the browser.
    • No matter what.
    • Even if you scroll, it hovers and stays there.
    • Try to imagine a really annoying advertisement that won't leave your screen. It's probably fixed.

And there you have it, the 4 key positioning elements! Hope that was helpful and clear, but if not, feel free to tell me where I'm wrong and where I can make things more clear.
