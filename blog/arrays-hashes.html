DBC Week 4: Arrays and Hashes

What are arrays and hashes?

This week we spent some time diving into the basics of arrays and hashes. While still early in our education, it's becoming increasingly clear that we'll need to understand both concepts, so here's a shot at explaining them. Before getting into differences, it's probably best to get into similarities (because you know...when you compare things, usually they share similarities).

Arrays and hashes are both used to store a bunch of objects together. For example, let's say we're storing fruits. One way we can do this is to use a variable, where we set fruit to save the string of "apple", like below:

 >> fruit = "apple"

This is cool, but in the event that we want to save multiple fruits, a variable isn't going to help too much with that. We could store multiple variables like below:

 >> fruit0 = "apple"
 >> fruit1 = "orange"
 >> fruit2 = "banana"

Feels kind of imcomplete, doesn't it? Here's where arrays come in. An array will store them as a collection, essentially grouped together. We could store as many fruits as we want.

 >> fruits = ["apple","orange","banana","grape","tomato"]

What's powerful about this is that we can retrieve items from the array, like this:

 >> return fruits[0]
 => "apple"
 >> return fruits[2]
 => "banana"

So how about hashes? Hashes will allow us to store even more information, for example, storing our fruit inventory. Hashes give us pairs of things we can store, where the first is the key and the second is the value.

 >> fruits = {apple: 3, orange: 2, banana: 1, grape: 50, tomato: 0}

...which then allows us to retrieve inventory like this:

 >> fruits[:apple]
 => 3
 >> fruits[:tomato]
 => 0

So what's really the difference?

Arrays and hashes in Ruby are both ordered, but one good way to think of an array is an ordered list, where the order is saved as the position they are in the array. In the example above, we looked for fruit #0 to get apple.

A hash on the other hand is also ordered, but only in the sense that the items stored in the hash are stored in the order we put them there. But we probably can't remember when we put apple in the hash, so we are simply looking to retrieve apple from the list. We can treat it as if it were unordered.

One great way I read about them is that an array is simply a hash, where the keys are always integers starting from 0.

When to use one vs. the other?

When thinking about which to use, I think of a few things:

  • If you only need to store a list and you don't need to remember where things are (don't care about order), an array will usually suffice.
  • If you actually need to retrieve specific items, a hash makes more sense.
  • Similarly, if you need each item to have a corresponding value, then a hash is definitely the way to go.

Well that's it! Go forth, use arrays and hashes wisely.
