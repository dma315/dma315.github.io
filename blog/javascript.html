DBC Week 7: JavaScript vs. Ruby

Iterating in Ruby

One of my favorite features of Ruby so far has been the simplicity of iterating through an array (or hash). We have a very powerful method .each that will go through every element in an array or hash. This is extremely powerful and we can do a lot of things with it. For example, we can add 2 to each element of an array and print it:

>>  array = [1, 2, 3, 4, 5]
>>  array.each { |i| p i + 2}
=>  3
=>  4
=>  5
=>  6
=>  7

Iterating in JavaScript

JavaScript isn't quite as nice to us. While jQuery (a popular JavaScript library) allows an each method, JavaScript itself doesn't appear to give us this nice native functionality. Instead, we have to essentially make our own each method, or iterate through every element of the array. We can do this using a for loop.

>>  var array = [1, 2, 3, 4, 5]
>>  for (var i = 0; i < array.length; i++) {console.log(array[i]+2})
=>  3
=>  4
=>  5
=>  6
=>  7

The nice thing here is that JavaScript provides a very simple way of writing for loops, but in this case, we have to manually define how many times we go through the loop (or iterate).

A breakdown of the for statement's components:

  1. Define an index: "For" a variable that we'll call i
  2. Set the iteration limit: Keep looping "while" i is less than the array's length (in this case 5). That is loop for i = 0, and keep going until i = 4 (but not 5, since indexes start at 0 in JS as well).
  3. Increment our index: For each loop, add 1 to i (i++ is just shorthand for i += 1, which is shorthand for i = i + 1. Programmers love shortcuts...

Conclusion

If there's one thing worth taking away here, it's that we can really take for granted how much easier our lives have been made by advancements in programming languages. Personally, I didn't really have a solid understanding of how and when to use for loops in Ruby, and learning JavaScript has helped solidify its purpose and what it does, namely break down something like the .each method.
